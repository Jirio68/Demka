Как сделать БД
CREATE DATABASE MyCompanyDB;

Как использовать нужную БД (переодически помогает при ошибках)
USE MyCompanyDB;

Как сделать таблицу
CREATE TABLE employees (
    id INT IDENTITY(1,1) PRIMARY KEY,
    name NVARCHAR(100) NOT NULL,
    position NVARCHAR(50),
    salary DECIMAL(10, 2)
);

Важно! N при создании столбца (к примеру NVARCHAR) позволяет заполнить столбец русским 

Как заполнить таблицу
INSERT INTO employees (name, position, salary) VALUES
(N'Иван Иванов', N'Программист', 85000.00),
(N'Ольга Петрова', N'Бухгалтер', 60000.00),
(N'Сергей Сидоров', N'Менеджер', 70000.00);

Важно! N перед заполнением необязательно, но тогда точно не будет никаких проблем с отображением русского

Как показать всю таблицу
SELECT * FROM employees;

Как сделать связь между таблицами
Часть 1:
	1) нужно сделать первую таблицу:
	
	CREATE TABLE departments (
		id INT IDENTITY(1,1) PRIMARY KEY,
		name NVARCHAR(100) NOT NULL
	);

Часть 2:
	1) создаём таблицу с внешним ключом departments_id
	
	CREATE TABLE employees (
    id INT IDENTITY(1,1) PRIMARY KEY,
    name NVARCHAR(100) NOT NULL,
    position NVARCHAR(50),
    salary DECIMAL(10, 2),
    department_id INT, -- внешний ключ

    FOREIGN KEY (department_id) REFERENCES departments(id)
);

Важно! 
FOREIGN KEY (department_id)	          -> "поле department_id в текущей таблице (например, employees) — это внешний ключ"
REFERENCES departments(id)            -> "department_id должен ссылаться на существующее значение id в таблице departments"

Как добавить столбец в таблицу
ALTER TABLE employees
ADD email NVARCHAR(100);

Важно!
employees                             -> название нужной таблицы (куда добавляем столбец)
email NVARCHAR(100)                   -> новый столбец, с типом данных (NVARCHAR) и количеством символов (100)

Как добавить запись в таблицу
Точно также как и заполнять (в самом вверху смотри)

Как обновить запись в таблице
UPDATE employees
SET salary = 90000.00
WHERE name = N'Иван Иванов';

Некоторые пояснения как работает:
UPDATE employees                      -> "я хочу изменить данные в таблице employees"
SET salary = 90000.00                 -> Обновляет поле salary (зарплата) и ставит новое значение 90000.00
WHERE name = N'Иван Иванов'	          -> Ограничивает обновление только для той строки, где поле name содержит текст Иван Иванов

N перед строкой	нужно для поддержки русских букв в SQL Server

Как сделать запрос на минимальное значение
SELECT MIN(salary) AS Минимальная_зарплата
FROM employees;

Как сделать запрос на максимальное значение
SELECT MAX(salary) AS Максимальная_зарплата
FROM employees;

Как сделать запрос на среднее значение 
SELECT AVG(salary) AS Средняя_зарплата
FROM employees;




Как сделать обязательную таблицу с пользователями

CREATE TABLE Пользователи (
    ID_Пользователя INT PRIMARY KEY IDENTITY(1,1),  -- Автоинкремент
    Логин NVARCHAR(50) NOT NULL,                    -- Строка (русские символы поддерживаются)
    Пароль NVARCHAR(255) NOT NULL,                  -- Строка (для хранения хешей паролей, если нужно)
    Роль NVARCHAR(20) NOT NULL                      -- Например: Админ, Пользователь
);

Основные типы данных:

INT	(Целое число)
NVARCHAR (строка с поддержкой русского)
DATE (2025-05-18)
TIME (14:30:00)
Как заполнять разные типы данных

INT	                                  -> без кавычек	123
NVARCHAR	                          -> Строка с N перед одинарными кавычками	N'Иван Иванов'
DATE	                              -> В формате ГГГГ-ММ-ДД, в кавычках	'2025-05-18'
TIME	                              -> В формате чч:мм:сс, в кавычках	'14:30:00'

3 нормальные формы (НФ) простым языком

1 НФ:
	"Одна ячейка — одно значение"

Игрушка	     Цвет

Машинка	     Красный
Машинка	     Синий
Мячик	     Жёлтый


2 НФ:
	"Все данные зависят только от главного"

Таблица "Игрушки"

Игрушка      Магазин

Машинка	     "Игрушки"
Кукла	     "Игрушки"
Мячик	     "Счастье"


Таблица "Магазины"

Магазин	     Адрес

"Игрушки"	 ул. Ленина, 10
"Счастье"	 пр. Победы, 5


Игрушка продаётся в магазине,
а магазин стоит по определённому адресу.
Значит, адрес магазина не зависит от игрушки,
а зависит только от самого магазина.


3 НФ:
	"Нет данных, которые зависят от чего-то кроме ключа"

Таблица "Магазины"

Магазин	     Адрес

"Игрушки"	 ул. Ленина, 10


Таблица "Продавцы"

Магазин	     Продавец

"Игрушки"	 Светлана


Если у тебя есть Продавец, который зависит от Магазина,
а Адрес тоже зависит от Магазина,
тогда Продавец и Адрес не должны быть в одной таблице.
